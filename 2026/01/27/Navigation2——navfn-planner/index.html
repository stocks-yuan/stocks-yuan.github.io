Navigation2——navfn_planner
最近更新：2026-02-04   |   字数总计：1.1k   |   阅读估时：4分钟   |   阅读量：次

navfn_planner 规划器没有考虑机器人运动学模型，将机器人简化一个点，适合用于全向移动型机器人。

零、效果展示

navfn_planner 规划器相比于 smac_planner2D 规划器的优势在于：

  1. navfn_planner 规划的路径相对平滑

  2. navfn_planner 在自由空间规划的路径更优，smac_planner2D 在自由空间规划的路径更倾向于走膨胀边缘

一、整体架构

nav2_navfn_planner 功能包采用两层架构：

  1. NavfnPlanner (navfn_planner.cpp/.hpp): ROS 2 插件接口层
    • 继承 nav2_core::GlobalPlanner ,负责与 Nav2 框架对接
    • 处理坐标转换、地图边界检查
  2. NavFn (navfn.cpp/.hpp): 核心算法层
    • 实现 Dijkstra/A* 势场计算
    • 纯数学运算，不依赖 ROS

二、核心工作流程

2.1 数据输入与准备

当 nav2_planner 调用 createPlan(start, goal) 时：

  • 锁定地图: 获取最新的全局代价地图（Global Costmap）。
  • 坐标转换:
    • 输入的 start 和 goal 是世界坐标系（米）。
    • NavfnPlanner 此时会计算它们在栅格地图中的 索引 (Map Index)。
      • 公式大致为：mx = (wx - origin_x) / resolution。
  • 配置算法内核:
    • 如果地图大小变了，它会重新调整内部数组的大小。
    • 它将 Costmap 的数据数组（unsigned char*）传递给 NavFn 对象。
    • 它设置起点和终点的索引。

2.2 势场传播 (Potential Field Propagation)

教科书上的 Dijkstra 更新公式：min(neighbor) + cost。但此方法生成的路径会倾向于走横向/纵向，存在明显的锯齿。

NavFn 采用二维波动方程，将规划的起点作为势场零点，像水波纹一样向四周或终点（使用A*时） 扩散。相比于 Dijkstra，双邻居插值会考虑波前是从 和 之间的某个方向 传来的，那么新势能 应该满足：

这是一个关于 的二次方程，解出来后用泰勒展开近似，就得到了下面的神秘公式的多项式系数。

  1. 初始化:

    • 将起点的势能设为 0。
    • 将所有其他点的势能设为无穷大。
    • 障碍物点的势能也是无穷大（不可通过）。

  2. 传播 (Propagation):

    • 算法会迭代地更新网格中每个点的势能值。
    • 一个点的势能 = 邻居中最小的势能 + 当前点的移动代价（Cost）。
    • 这个过程像波纹一样从起点向外扩散，直到覆盖到终点（或者覆盖全图）。
    • 代价地图中的灰色区域（Cost > 0）会增加传播的“阻力”，使得势能更高。

注：这就是为什么它叫 Navigation Function (NavFn)，它计算了一个标量场函数。

2.3 梯度下降生成路径

一旦势场计算完成，整个地图就有了一个势场图，路径规划的起点为零势能点，生成路径的过程就是从 终点(某一高势场点) 开始，顺着势场下降最快的方向（梯度方向）往回滑，直到滑到起点。
