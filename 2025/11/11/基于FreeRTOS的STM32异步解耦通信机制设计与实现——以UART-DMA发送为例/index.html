基于FreeRTOS的STM32异步解耦通信机制设计与实现—以UART DMA发送为例
最近更新：2026-04-12   |   字数总计：2k   |   阅读估时：10分钟   |   阅读量：次

Github 项目：stm32-freertos-async-uart

一、引言

在嵌入式实时系统中，外设通信（如UART、SPI、I2C）通常面临以下挑战：

  • 阻塞式调用导致CPU利用率低

  • 多任务竞争引发数据损坏

  • 高耦合性使得代码难以维护和扩展

本文以STM32H7的UART DMA通信为例，提出一种基于FreeRTOS的​​异步解耦通信框架​​，通过消息队列、二值信号量、互斥量管理，实现高效可靠的外设数据传输。做到了各业务线程只管发数据而不用等，外设线程自动排队发出数据包，业务层和底层驱动随时可以替换，系统运行效率更高。

技术点：

  • 解耦（面向对象）：通过 C 语言结构体与函数指针（如 UART_DEVICE_T）封装底层，向应用层暴露标准 API。

  • 异步（IPC机制）：使用消息队列分离业务层数据请求与 底层DMA 实际发送，使用二值信号量同步发送任务与 DMA 中断，使用互斥量保护外设公共资源 UART_DATA_T。

三、异步通信机制设计

这里主要使用了3种FreeRTOS中的IPC通信机制：

  • 消息队列：解耦生产者（数据请求发送任务）与消费者（DMA发送任务），避免任务中直接调用导致阻塞

  • 二值信号量：实现串口发送任务与发送完成中断之间的同步，避免无同步连续调用DMA发送函数导致发送数据覆盖问题。

  • 互斥量：避免多线程任务争夺唯一外设资源，保护UART_DATA_T中的数据，防止数据竞争。

3.1 发送接口

stm32_uart_dma_send函数没有直接调用HAL库发送函数，而是通过消息队列将消息发送到外设发送任务中处理。uart_send_task_handle任务负责读取消息队列信息然后调用HAL_DMA发送，队列中没有消息时，此任务就会无限期阻塞。uart_data->xTxSem二值信号量同步外设发送任务与发送完成中断，避免了DMA的数据覆盖问题。

通过这种异步通信的设计，可以减少其他任务在调用外设函数时阻塞等待的时间，增大CPU利用率。
